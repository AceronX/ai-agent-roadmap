## Resource Quality Bar

- Prefer official documentation, canonical papers, and well-maintained projects.
- Add a short note explaining why the resource is useful.
- Avoid low-quality listicles, outdated tutorials, and untested code dumps.
- Put topic-specific resources in the relevant stage topic folder.
